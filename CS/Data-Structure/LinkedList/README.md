# LinkedList 직접 구현해보기

## 연결리스트
연결리스트란, 각 노드가 데이터와 포인터를 가지고 한줄로 연결되어 있는 방식의 자료구조를 말합니다. 데이터를 담고 있는 노드들이 연결되어 있고, 노드의 포인터가 이전, 다음 노드와의 연결을 담당합니다.

데이터를 추가/삭제를 할 때는 노드의 참조만 바꾸어주면 되어 추가/삭제는 배열에 비해서 용이하지만, 인덱스가 없는 리스트의 특징으로 인하여 특정 요소에 접근하기 위해서는 노드를 순차적으로 탐색해야하기 때문에 **탐색 속도가 떨어집니다.**

데이터의 추가/삭제가 많은 경우에는 연결리스트를 사용하고, 탐색이나 정렬을 자주하는 경우에는 배열을 사용하는것을 권장합니다.

※자바의 LinkedList의 경우 인덱스를 사용하여 연산을 수행할 수 있도록 get(index) 형태의 메서드를 제공하지만, 메서드 내부 동작은 순차 탐색으로 이루어져 있습니다.

연결 리스트의 종류는 3가지가 있는데, 다음과 같습니다.

1. 단일 연결 리스트(Singly Linked List)
    - 각 노드마다 다음 노드에 대해서만 참조하는 가장 단순한 형태의 연결리스트입니다.
    - 큐를 구현할 때 사용됩니다.
    - Head 노드를 잃어버려 참조하지 못하는 경우 데이터 전체를 사용 못하게 되는 단점이 있습니다.

2. 이중 연결 리스트(Doubly Linked List)
    - 각 노드마다 이전 노드, 다음 노드에 대해서 참조하는 형태의 연결 리스트입니다.
    - 삭제가 간편합니다(각 노드마다 이전,다음에 대해서 알기때문)
    - 단일 연결리스트에 비해 데이터 손상에는 강하지만, 관리할 참조가 이전과 다음 두가지이기 때문에 삽입이나 정렬을 할 경우 해야할 일이 더욱 많습니다.

3. 원형 연결 리스트(Circular Linked List)
    - 연결 리스트에서 마지막 요소가 첫번째 요소를 참조하여 원형으로 연결되어지는 리스트입니다.
    - 스트림 버퍼의 구현에 많이 사용됩니다.
