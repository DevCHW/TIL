<summary>객체지향 프로그래밍(OOP)이란?</summary>
<div>

```
우리가 실생활에서 쓰는 모든 것을 객체라 하며, 객체 지향 프로그래밍은 프로그램 구현에 필요한 객체를 파악하고 상태와 행위를 가진 객체를 만들고 각각의 객체들의 역할이 무엇인지를 정의하여 객체들간의 상호작용을 통해 프로그램을 만드는 것을 말합니다.  

즉, 기능이 아닌 객체가 중심이며 "누가 어떤 일을 할 것 인가?" 가 핵심입니다.  

객체 지향 프로그래밍의 특징으로는 캡슐화, 상속, 다형성, 추상화 등이 있고, 모듈 재사용으로 확장 및 유지보수가 용이합니다.
```

<details>
<summary>불변 객체나 final을 굳이 사용해야 하는 이유?</summary>
<div>

```
1. `Thread-Safe` 하여 병렬 프로그래밍에 유용하며, 동기화를 고려하지 않아도 된다.

2. 실패 원자적인 메소드를 만들 수 있다.(어떠한 예외가 발생되더라도 메소드 호출 전의 상태를 유지할 수 있어 예외 발생 전과 똑같은 상태로 다음 로직 처리 가능)

3. 부수효과를 피해 오류를 최소화 할 수 있다.

### 부수효과란?
- 변수의 값이 바뀌거나 객체의 필드 값을 설정하거나 예외나 오류가 발생하여 실행이 중단되는 현상

4. 메소드 호출 시 파라미터 값이 변하지 않는다는 것을 보장할 수 있다.

5. 가비지 컬렉션 성능을 높일 수 있다.
(가비지 컬렉터가 스캔하는 객체의 수가 줄기 때문에 GC 수행 시간도 줄어든다.)

요약
1. Thread-Safe
2. 호출한 메소드가 실패하더라도 상태유지 가능
3. 오류 최소화
4. 메소드 호출 시 파라미터 값이 불변하다는 것을 보장
5. 가비지 컬렉션 성능 증가

```
</div>

<summary>추상 클래스와 인터페이스에 대한 설명과 차이점</summary>
<div>

```
추상클래스는 클래스 내 추상 메소드가 하나 이상 포함되거나 abstract로 정의된 경우를 말하고, 인터페이스는 모든 메소드가 추상 메소드로만 이루어져 있는 것을 말합니다.  

공통점은 new 연산자로 인스턴스를 생성할 수 없다는 것과 사용하기 위해서 하위 클래스에서 확장 또는 구현해야 한다는 점이 있습니다.  

차이점은 인터페이스는 그 인터페이스를 구현하는 모든 클래스에 대해 특정한 메소드가 반드시 존재하도록 강제함에 있고, 추상 클래스는 상속받는 클래스들의 공통적인 로직을 추상화시키고, 기능 확장을 위해 사용한다는 점이 있습니다. 또, 추상 클래스는 다중상속이 불가능하지만, 인터페이스는 다중 상속이 가능하다는 점도 있습니다.

요약

공통점  
1. new 인스턴스로 생성 X
2. 사용하려면 하위 클래스에서 확장 또는 구현해줘야 함  

차이점
1. 인터페이스는 구현 클래스에서 인터페이스의 모든 메소드를 반드시 구현해주어야 하지만, 추상클래스는 기능 확장을 위해 사용됨
2. 인터페이스는 다중상속 가능, 추상클래스는 다중상속 불가능
```
